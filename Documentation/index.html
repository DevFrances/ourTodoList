Introduction: Getting started

The TodoApp helps a user to generate a list of items, filter through, and delete list items from the log.

The app also marks every Todo as active or completed, and displays only all, active or completed Todo list.

It also clears all completed Todos from the list array.

The design is programmed with pure JavaScript also known as Vanilla JavaScript.

Technical Guide

The todo-list-app is built using the MVC - Model, View, Controller blueprint

The Model, View and the Controller are separate entities. The Controller serves as the intermidiary between the Model and the View, thereby making it the only entity that can interact with the both.

The Model View Controller

  • Model: Model is responsible for managing the data of the application.
  • View: View is a presentation of the model in a particular format - in our case displaying All, Active or Completed todos (route). It also let user to interact with displayed data.
  • Controller: Controller responds to the user input from View and performs interactions with Model.

  • store.js

    Bug which may leads to potential conflict between duplicate IDs

    bug:

    for (var i = 0; i < 6; i++) { newId += charset.charAt(Math.floor(Math.random() * charset.length)); }

    Solved:

    var isUnique = false; while (!isUnique){ for (var i = 0; i < 6; i++) { newId += charset.charAt(Math.floor(Math.random() * charset.length)); } isUnique = true; for (var i = 0; i < todos.length; i++) { if (todos[i].id == newId) { isUnique = false; } } }

  • index.html line 16

    Missing id in input tag for toggle-all label which prevent toggle all todos to work properly.

    <input class="toggle-all" type="checkbox"> <label for="toggle-all">Mark all as complete</label> Fixed: <input class="toggle-all" id="toggle-all" type="checkbox">
